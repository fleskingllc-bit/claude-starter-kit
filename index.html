Chapter 1

はじめに — Claude の3つのモード

1.1 このマニュアルについて

このマニュアルは、AIエージェント「Claude」を確実に導入するためのガイドです。

Claude には3つの使い方（モード）があり、あなたの役割に合った方法で導入します。各ステップでURLをクリックし、コマンドをコピー＆ペーストするだけで完了します。

途中で困った場合は、セクション番号（例: 3A.2）を控えてサポートにご連絡ください。

1.2 Claude の3つのモード

Chat — 会話する

ブラウザやアプリで Claude と対話します。質問に答えてもらったり、文章を作ってもらったり。
最も基本的なモードです。全員が使います。

Cowork — 納品してもらう

Claude に仕事を「丸ごと任せる」モードです。
たとえば「このフォルダの資料から経営会議用のスライドを作って」と指示すると、あなたがPCから離れている間に作業を進めて、完成品を納品してくれます。

Chat との違い: Chat は「会話のキャッチボール」。Cowork は「仕事を渡して、できあがりを受け取る」。

Code — 開発する

ターミナル（黒い画面）で Claude に指示を出し、プログラムやツールを開発するモードです。
社内ツール・自動化の仕組み・Webサービスなどを構築できます。開発担当者が使います。

Chapter 2

Claude アカウント作成

2.1 アカウント登録

以下のURLを開いてください。

アクセス先
https://claude.ai/login

「Sign up」（新規登録）をクリックします。

すでにアカウントをお持ちの場合は「Log in」からログインし、2.2 へ進んでください。

登録方法を選びます。

方法手順おすすめ度
Google アカウント「Continue with Google」をクリック★★★
メールアドレスメール入力 → 確認コード入力★★
Google アカウントでの登録を推奨します。パスワード管理が不要で最も簡単です。

電話番号を認証します。携帯番号を入力 → SMSで届く6桁のコードを入力。

利用規約に同意して完了。これで無料版の Claude が使えます。

2.2 Pro プラン契約

Cowork モードと Claude Code は、Pro プラン以上が必要です。
月額 $20（約3,000円）。Chat のみの利用でも、応答回数の上限が大幅に増えます。

以下のURLを開きます。

アクセス先
https://claude.ai/settings/billing

「Upgrade to Pro」をクリック → カード情報を入力 → 「Subscribe」で完了。

決済は米ドル（USD）です。日本のカードでも利用可能です。

同じアカウントを複数デバイスで使えます。
会社のPC、自宅のPC、スマホなど、どこからでもログインして利用できます。ただし利用量の上限はアカウント全体で共有されます。

Chapter 3A

Claude Desktop インストール

Claude Desktop は、Chat と Cowork が使えるアプリです。全員がインストールしてください。
開発者の方も、まずこちらを完了してから 3B章 に進んでください。

3A.1 アプリのダウンロード

以下のURLを開いてください。

ダウンロード
https://claude.ai/download

お使いのOSに合ったボタンをクリックしてダウンロードします。

  • Mac: .dmg ファイルを開き、アプリケーションフォルダにドラッグ
  • Windows: .exe ファイルを実行してインストール

アプリを起動します。

Dock（Mac）またはスタートメニュー（Windows）から「Claude」を起動します。

3A.2 初回ログイン

2章で作成したアカウントでログインします。

Google アカウントで登録した場合は「Continue with Google」をクリック。

ログイン完了後、Claude のチャット画面が表示されます。

3A.3 Chat モードで動作確認

メッセージ入力欄に日本語で話しかけてみましょう。

> こんにちは。あなたは何ができますか？日本語で簡単に教えてください。

日本語で回答が返ってくれば正常動作です。

3A.4 Cowork モードを試す

Cowork は Mac（Apple Silicon: M1以降）と Windows で利用可能です。
古い Mac（Intel）では利用できません。

Claude Desktop の画面上部で「Cowork」モードを選択します。

Chat と Cowork の切り替えボタンが表示されています。

簡単なタスクを試してみましょう。

> デスクトップに「テスト.txt」というファイルを作成して、「Claude導入テスト成功」と書いてください。

Claude が確認を求めたら「Allow」（許可）をクリック。デスクトップにファイルができれば成功です。

3A章完了! Claude Desktop のインストールと動作確認ができました。
ビジネスユーザーの方は 4章 へ進んでください。開発者の方は 3B章 へ。
開発者のみ — ビジネスユーザーはスキップ → 4章へ
Chapter 3B

Claude Code 有効化

この章は開発者（タイプB）だけが読みます。
ビジネスユーザー（タイプA）の方は、このページを飛ばして 4章 Google Drive 導入 に進んでください。

Claude Code は、PCのファイルやフォルダを直接操作できるAIエージェントです。プログラム開発、ツール構築、自動化に使います。
3A章でインストールした Claude Desktop の中に含まれています。追加インストールは不要です。

3B.1 Code タブの有効化

Claude Desktop を開きます。

設定画面を開きます。

Mac: メニューバー「Claude」→「Settings...」（または Command + ,）
Windows: 左下の歯車アイコン → 「Settings」

「Claude Code」の項目を見つけて、有効化（ON）にします。

有効化すると、画面上部に「Code」タブが追加されます。
Chat / Cowork に加えて、Code タブからファイル操作やプログラム実行ができるようになります。

3B.2 動作確認テスト

「Code」タブをクリックして切り替えます。

日本語で話しかけてみましょう。

> こんにちは。今日の日付を教えてください。

日本語で日付が返ってくれば正常動作です。

ファイル作成テスト:

> デスクトップに「テスト.txt」を作成して「Claude Code導入テスト成功」と書いてください。

Claude が「許可しますか？」と聞いてきたら「Allow」で進めてください。デスクトップにファイルができれば成功です。

3B.3 ターミナル版について（上級者向け）

通常はDesktop内のCodeタブだけで十分です。以下のような高度な用途が必要になった場合のみ、ターミナル版（CLI）の追加インストールを検討してください。
用途説明
スクリプト自動実行定期的なバッチ処理をPCに任せたい場合
エージェントチーム複数のAIインスタンスを連携させる場合
CI/CD連携GitHubと連動した自動テスト・デプロイ

ターミナル版が必要になったときは担当者（谷本）にご相談ください。

3B章完了! Claude Code の有効化と動作確認ができました。4章へ進んでください。

Chapter 4

Google Drive デスクトップ 導入

Google Drive デスクトップアプリを入れると、クラウド上のファイルをPC上のフォルダとして扱えます。
ここで作る Obsidian フォルダが、今後の Claude の作業場所になります。

なぜ Google Drive 上に作るの？
PCが壊れてもデータが失われません。複数デバイスからアクセスでき、同じ会社のメンバーとも安全にファイルを共有できます。

4.1 アプリのインストール

以下のURLを開いてください。

ダウンロード
https://www.google.com/drive/download/

「パソコン版ドライブをダウンロード」をクリック。

ダウンロードされたファイルを開いてインストールします。

  • Mac: .dmg ファイルを開き、アプリケーションフォルダにドラッグ
  • Windows: .exe ファイルを実行してウィザードに従う

4.2 ログインと同期設定

アプリを起動し、Google アカウントでログインします。

同期設定はデフォルトのままでOK。

設定完了後、以下の場所に Google Drive フォルダが現れます:

OSパス
Mac~/Library/CloudStorage/GoogleDrive-（メールアドレス）/マイドライブ/
WindowsG:\マイドライブ\ またはエクスプローラーの「Google Drive」

4.3 Obsidian Vault フォルダの作成

Obsidian（オブシディアン）って何？
メモやドキュメントを管理する無料アプリです。次の5章で詳しく設定しますが、今はフォルダだけ作っておけばOKです。深く考えなくて大丈夫！

Google Drive の「マイドライブ」内に、新しいフォルダを作ります。

  • 方法A: Finder（Mac）/ エクスプローラー（Windows）で Google Drive フォルダを開き、右クリック → 新規フォルダ
  • 方法B: drive.google.com をブラウザで開き、「新規」→「フォルダ」

フォルダ名を Obsidian_Vault にします。

これはただの普通のフォルダです。特別なアプリではありません。
このフォルダの中に、Claudeが作るメモや資料が保存されていきます。名前は「Obsidian_Vault」で統一してください（後の設定で使います）。

Chapter 5

Obsidian 導入と初期設定

Obsidian は、マークダウン形式でノートを管理する無料アプリです。Claude が作成するドキュメントの閲覧・編集に使います。

5.1 Obsidian のインストール

以下のURLを開いてダウンロードします。

ダウンロード
https://obsidian.md/download

お使いのOSに合ったバージョンをダウンロードしてインストールしてください。

Obsidian は完全無料で使えます。有料プランへの加入は不要です。

5.2 Vault（保管庫）の設定

Obsidian を起動します。

「Open folder as vault」（フォルダを保管庫として開く）を選択。

4.3 で作成した Obsidian_Vault フォルダを選択します。

Mac~/Library/CloudStorage/GoogleDrive-.../マイドライブ/Obsidian_Vault
WindowsG:\マイドライブ\Obsidian_Vault

5.3 フォルダ構成について

Obsidian Vault 内は、以下のフォルダ構成でAIが自動で整理・管理・作成していきます。

Obsidian_Vault/ ├── 00_inbox/ ← 未整理・一時メモ ├── 10_projects/ ← プロジェクト別フォルダ ├── 20_areas/ ← 継続的な業務領域 ├── 30_resources/ ← 参考資料・ナレッジ ├── 40_archive/ ← 完了・アーカイブ └── templates/ ← テンプレート
手動で作る必要はありません！次の6章で Claude に初回セットアップを実行してもらうと、このフォルダ構成とテンプレートを自動で作成してくれます。ここでは「こういう構成になるんだな」と眺めるだけでOKです。
5章完了! Obsidian の環境が整いました。次章で Claude と連携させます。

Chapter 6

Claude 初期設定

Claude をあなたの業務環境に合わせてカスタマイズします。
担当者（谷本）から受け取る「スターターキット」を使って設定します。

6.1 スターターキットのダウンロード

担当者（谷本）から共有される GitHub のURLを開きます。

ダウンロード先
https://github.com/fleskingllc-bit/claude-starter-kit

緑色の「Code」ボタン →「Download ZIP」をクリックしてダウンロードします。

ダウンロードフォルダに ZIP ファイルが保存されます。ダブルクリックで解凍してください。

スターターキットには以下が含まれています:

ファイル役割
CLAUDE.mdClaude への指示書（会社名・ルール等）
.claude/settings.json権限設定ファイル
.claude/skills/wrap/作業記録スキル（6.5で説明）
first-session-prompt.md初回セットアップ用プロンプト
ファイルの配置は手動で行う必要はありません。次のステップで Claude が自動で正しい場所に配置してくれます。

6.2 CLAUDE.md とは

CLAUDE.md は、Claude に「あなたは誰で、何をしてほしいか」を伝える指示書です。
ここに書いたルールを Claude は常に守ります。

CLAUDE.md に書かれている内容

項目内容
基本情報会社名、あなたの名前、役職、業種
作業ルール日本語で回答、変更前に確認を取る、バックアップを取る
提案ルール成功確度（高/中/低）を必ず付記、費用が発生する場合は事前明示
禁止事項ファイル削除禁止（アーカイブのみ）、メール送信禁止（下書きまで）
ファイル管理Obsidian Vault のパス、フォルダ構成のルール
サポート連絡先谷本の連絡先
CLAUDE.md は後からいつでも編集できます。使っていく中で「こうしてほしい」と思うことがあれば、Claude に「CLAUDE.md にこのルールを追加して」と伝えるだけです。

6.3 Claude に初期設定してもらう

ここからが「Claude の魔法」です。プロンプトを1つ貼るだけで、Claude がすべてを自動で設定します。

ビジネスユーザーの方

Claude Desktop を開き、Cowork モードで実行してください。

開発者の方

Claude Code（ターミナル）で実行してください。

以下のプロンプトをすべてコピーして、Claude に貼り付けて送信します。

ダウンロードフォルダにスターターキット（starter-kit）があります。 以下の手順で初期設定を行ってください。各ステップで何をするか説明してから実行してください。 1. ダウンロードフォルダでスターターキットを見つけてください（starter-kit フォルダまたは ZIP を解凍したフォルダ） 2. キット内のファイルを正しい場所に配置してください: - CLAUDE.md → ホームフォルダ直下（~/CLAUDE.md） - .claude/ フォルダ → ホームフォルダ直下（~/.claude/） ※ 既に .claude/ が存在する場合は、中身をマージしてください（既存ファイルは上書きしない） 3. ホームフォルダに配置した CLAUDE.md を読み込み、「基本情報」セクションの未入力項目を私に質問して埋めてください: - 会社名 - 担当者（氏名） - 役職 - 業種 4. PCのクラウドストレージを自動検出してください: - ~/Library/CloudStorage/ （Mac）または ~/OneDrive/ や ~/Google Drive/ を探す - Google Drive、OneDrive、その他のクラウドストレージのどれが使われているか特定する - 見つかったクラウドストレージ内に Obsidian_Vault フォルダがなければ作成する - CLAUDE.md の「ファイル管理」セクションのパスを、検出したクラウドストレージのパスに自動修正する - クラウドストレージが見つからない場合は、どのサービスを使っているか質問する 5. Obsidian Vault 内に以下のフォルダ構成を作成してください（既にあればスキップ）: - 00_inbox/ - 10_projects/ - 20_areas/ - 30_resources/ - 40_archive/ - templates/ 6. スターターキットの templates/ フォルダ内のテンプレートファイルを Obsidian Vault の templates/ にコピーしてください: - meeting-note.md（会議メモ用） - project-note.md（プロジェクトノート用） - daily-note.md（日次メモ用） 7. 設定内容を要約して報告してください

Claude があなたに質問しながら、以下をすべて自動で行います:

  • ダウンロードフォルダからファイルを正しい場所に配置
  • CLAUDE.md の基本情報（会社名・名前・役職・業種）をヒアリングして入力
  • 権限設定ファイルの配置
  • クラウドストレージ（Google Drive / OneDrive 等）を自動検出
  • Obsidian Vault フォルダの確認・作成、パスを自動設定
  • Obsidian Vault 内のフォルダ構成を自動作成（5.3 で紹介した構成）
  • テンプレートファイル（会議メモ、プロジェクトノート、日次メモ）を自動作成
  • 設定結果の要約レポート

Claude の質問に答えていくだけで、すべての初期設定が完了します。

Claude が「許可しますか？」と聞いてきたら「はい」（Allow）で進めてください。
ファイルの配置や作成のたびに確認を求めてきます。内容を確認して許可してください。

6.4 設定内容の確認

Claude が初期設定を完了すると、以下が自動で配置・作成されています:

項目場所内容
CLAUDE.mdホームフォルダ直下あなたの情報が入力済みの指示書
.claude/settings.jsonホームフォルダ直下安全な権限設定
.claude/skills/wrap/ホームフォルダ直下作業記録スキル
フォルダ構成Obsidian Vault 内00_inbox〜40_archive, templates
テンプレートObsidian Vault/templates/会議メモ、プロジェクトノート、日次メモ
権限設定について:Claude はファイルの読み取りと検索は自由にできますが、書き込みやコマンド実行は毎回あなたの許可を求めます。安全な初期設定です。使い慣れたら設定を緩めることもできます。

6.5 便利な機能:「/wrap」で作業記録

スターターキットには /wrap という便利な機能（スキル）が含まれています。

/wrap とは？

Claude との作業が終わったときに /wrap と入力するだけで、今回のセッションで何をしたかを自動的にまとめて Obsidian に保存してくれます。

手書きの作業メモが不要になります。後から「あのとき何をやったっけ？」と振り返るのに便利です。

かならず、ひとまとまりの作業が終わったら実行して、AIの長期記憶を作りましょう！

使い方

Claude との作業がひと区切りついたら、以下を入力します。

> /wrap

Claude が自動で以下を行います:

  • 今回のセッションで何をしたかを振り返る
  • 作業内容・変更点・今後の予定を整理する
  • Obsidian Vault 内に作業ノートとして保存する

保存されたノートを Obsidian で確認できます。

保存されるノートの例

# Slack Bot 初期設定 作業日: 2026-03-15 ## 概要 社内連絡用の Slack Bot を作成し、 基本的なメッセージ応答機能を実装した。 ## 作業内容 ... ## 今後の予定 - 議事録自動生成機能の追加 - スケジュール通知の設定
毎回の作業の最後に /wrap を習慣にしましょう。
チームメンバーが Obsidian を開けば、誰がいつ何をしたかが一目でわかります。
6章完了! すべての初期設定が完了しました。

Chapter 7

トラブルシューティング

7.1 よくある問題と解決法

7.1.1 Claude Desktop が起動しない

症状対処法
アプリが開かないPCを再起動してから再度試す
ログイン画面が出ないアプリを削除して再インストール（4A.1）
Cowork が選べないMac: Apple Silicon（M1以降）が必要。左上 → このMacについて で確認

7.1.2 Claude Code のインストールでエラーが出る（開発者向け）

症状対処法
「command not found: curl」Mac: ターミナルで xcode-select --install を実行
「Permission denied」管理者権限がない可能性。IT部門に確認。
ネットワークエラーWi-Fiを確認。VPN を使用中なら一時的にOFFに。

7.1.3 claude コマンドが見つからない（開発者向け）

ターミナル / PowerShell を一度閉じて再度開いてください。
それでも解決しない場合:

# Mac $ ls ~/.local/bin/claude # Windows PS> ls $env:USERPROFILE\.local\bin\claude.exe

7.1.4 「Pro plan required」と表示される

セクション 2.2 に戻って Pro プランを契約してください。

7.1.5 Google Drive が同期されない

タスクバー / メニューバーの Google Drive アイコンをクリックして同期状態を確認。
「一時停止」になっていたら再開してください。

7.1.6 Obsidian でファイルが見えない

Google Drive の同期が追いついていない可能性があります。数分待ってから Obsidian を再起動してください。

7.2 診断コマンド

解決しない場合は、以下の情報をスクリーンショットで撮ってサポートに送ってください。

Claude Desktop の場合

アプリ内の設定画面 → 「About」のバージョン情報をスクリーンショット

Claude Code の場合（開発者向け）

$ claude doctor

Chapter 9

初期設定ファイルで始める

このチャプターは初期設定ファイル（ZIP）を受け取った方向けです。
メールで届いたZIPファイルを使って、あなたの会社専用の環境を構築します。

9.1 ZIPファイルの展開

Step 1: ZIPファイルをダウンロード

メールに添付された hikari-ai-setup_○○会社.zip をダウンロードしてください。
通常は ダウンロード フォルダに保存されます。

Step 2: ダブルクリックで展開

ZIPファイルをダブルクリックするとフォルダが展開されます。それだけでOKです。

中に以下のファイルが入っています:

CLAUDE.md ← あなたの会社専用の設定ファイル onboarding.md ← 導入チェックリスト（AIが案内役） commands/ ← スキルファイル（Claude が自動で配置します） plans/ ← 業務構築計画書（選択した項目のみ）
フォルダの移動やコピーは不要です。展開した場所でそのまま使えます。

9.2 Claude Desktop で開く

Step 3: Claude Desktop アプリを開く

Chapter 3A でインストールした Claude Desktop アプリを起動します。

Step 4: 展開したフォルダをプロジェクトとして開く

Claude Desktop の画面で、展開したフォルダ（hikari-ai-setup_○○会社）をプロジェクトとして開きます。

開き方: Claude Desktop の左メニュー → プロジェクト → フォルダを追加 → 展開したフォルダを選択

Step 5: オンボーディングを開始

Claude に以下のように入力してください:

onboarding.md を読んで、次のステップを教えて

Claude が onboarding.md を読み、あなたの導入状況に合わせて次にやるべきことを案内してくれます。

スキルファイルの配置なども Claude が自動で行います。ターミナル操作は不要です。

以降は Claude の指示に従って進めるだけです。

9.3 業務構築の始め方

オンボーディングが Phase 4 まで進むと、plans/ フォルダ内の構築計画書を使う段階になります。

構築計画書の使い方

例えば売上管理を構築したい場合:

plans/sales_plan.md を読んで、これを元に構築を始めて

Claude があなたに質問しながら、最適な仕組みを一緒に作っていきます。
事前準備は不要です。Claude が必要な情報をヒアリングします。
