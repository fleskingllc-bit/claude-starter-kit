# セッション作業まとめ → Obsidian 保存

## 目的
今回の会話セッションで行った作業内容を、Obsidian Vault にマークダウンノートとして保存する。
後から見返したときに「何をやったか」「何が変わったか」「次に何をするか」がわかるようにする。

## 手順

### 1. 会話の振り返り
今回のセッションで行った作業を、会話の流れから整理する:
- **何をしたか**（調査、資料作成、設定変更、ツール導入など）
- **変更したファイル・設定**（ドキュメント、設定、外部サービスなど）
- **判断・決定事項**（なぜその方法を選んだか）
- **発生した問題と解決方法**
- **残タスク・今後の予定**

### 2. 保存先の決定
引数 `$0` またはセッション内容から適切なフォルダを判定する。

Obsidian Vault のパスは CLAUDE.md の「ファイル管理」セクションに記載されている。
見つからない場合は Google Drive 内の `Obsidian_Vault/` を検索する。

保存先フォルダの選び方:

| 内容 | フォルダ |
|---|---|
| 進行中のプロジェクトに関する作業 | `10_projects/{プロジェクト名}/` |
| 継続的な業務（経理、営業等）の作業 | `20_areas/` |
| 調べ物・参考情報のまとめ | `30_resources/` |
| 分類に迷うもの | `00_inbox/` |

### 3. ファイル名
`{内容の要約}.md`
- 例: `Slack-Bot初期設定.md`
- 例: `月次レポート自動化-調査.md`
- 日本語OK、スペースはハイフンに

### 4. ノートの構成

```markdown
# {タイトル}

作業日: {YYYY-MM-DD}

## 概要
{1〜3行で何をしたか}

## 作業内容
### {作業1のタイトル}
{内容}

### {作業2のタイトル}
{内容}

## 変更したもの
{変更ファイル、設定、導入したツールなど。テーブル形式推奨}

## 発生した問題と対応
{あれば。なければセクション省略}

## 今後の予定
{残タスク、次にやること。なければセクション省略}
```

### 5. 注意事項
- **簡潔に書く**: 詳細なコードは書かず、要点だけまとめる
- **判断の理由を残す**: 「なぜその方法にしたか」は後で重要になる
- **パスワードやAPIキー等の秘密情報は絶対に書かない**
- **既存ノートがある場合**: 同じテーマのノートがあれば、新規作成ではなく追記するか判断する
- 保存後、ファイルパスをユーザーに表示する
